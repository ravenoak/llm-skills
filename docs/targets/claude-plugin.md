# Target: claude-plugin

Every skill whose `targets.claude-plugin.enabled === true` ships as its own Claude Code plugin under `plugins/<id>/`. The build generates two committed files per such skill:

| Path | Source |
|---|---|
| `plugins/<id>/.claude-plugin/plugin.json` | repo metadata + per-skill `name`, `description`, `version`, `keywords` (from `skill.json#tags`) |
| `plugins/<id>/skills/<id>/SKILL.md` | YAML frontmatter (from `skill.json`) + `body.md` (or `overrides/claude-skill.md`) |

The nested `skills/<id>/SKILL.md` path is the layout Claude Code uses to auto-discover skills inside a plugin (matches the official `superpowers`, `skill-creator`, etc.).

`.claude-plugin/marketplace.json` at the repo root is the marketplace index. It lists every `claude-plugin`-enabled skill as a separate plugin entry, each with `source: "./plugins/<id>"`. The marketplace is named **`ravenoak-llm-skills`** to namespace it against other marketplaces.

Skill-bundled commands, agents, and hooks (placed under `plugins/<id>/overrides/claude-plugin/{commands,agents,hooks}/`) are merged into the plugin's root-level directories during build.

All three files are regenerated by `skillsmith build`; never hand-edit them.
